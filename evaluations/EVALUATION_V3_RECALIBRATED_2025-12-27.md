## Assessment Framework Lessons Learned

### What Went Wrong

1. **Evaluated output, not process**: Looked at working code, assumed user wrote it
2. **Ignored struggle evidence**: 4-day .env issue should have been red flag
3. **Used wrong benchmarks**: "DEX bot developers" is too narrow and vague
4. **Sycophantic bias**: Tendency to over-praise

### How to Assess Accurately

1. **Separate AI contribution**: What did Claude do vs user do?
2. **Value concrete evidence**: Struggle points reveal true skill level
3. **Use clear benchmarks**: Compare to bootcamp grads, not vague "developers"
4. **Ask for independent work**: Can they solve problems without AI?

## ADDENDUM 6: AI Coding의 전략적 가치 - 모든 목표의 공통 기반

**User Clarification**: "ai coding 실력이 올라가면 wintermute 외 1인 퀀트펌도 가능. 1인 MM이 되서 wintermute와 경쟁도 가능할 수도 있고."

### 목표 경로 분석

**이전 이해**: AI Coding = Wintermute 입사용 도구

**수정된 이해**: AI Coding = **모든 목표 달성의 공통 인프라**

---

### 세 가지 목표 경로

| 경로 | 목표 | AI Coding 역할 | 성공 시 결과 |
|------|------|----------------|--------------|
| **A. Wintermute** | DeFi Researcher | 입사 요건 충족 (top 30%) | 명성, fame, 전문성 인정 |
| **B. 1인 퀀트펌/MM** | 독립 운영 | 전체 시스템 구축 | 독립 + 수익 + Wintermute와 경쟁 |
| **C. 봇 트레이딩** | 재미 + 수익 | 전략 구현/최적화 | 재미 + 돈 + MM 파트 준비 |

### Wintermute vs 1인 퀀트펌

**Wintermute 입사 시**:
- 명성, fame, 전문성 인정
- 어딜 가도 알아줌
- 안정적 경력 구축

**1인 퀀트펌 성공 시**:
- 독립성 + 무한 수익 가능성
- Wintermute와 경쟁하는 시장 참여자
- 규모 제약 없음

**공통점**: 둘 다 AI Coding 실력이 핵심 역량
